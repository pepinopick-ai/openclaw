## Режим 1 — Контроль партии

```
Для проверки партии запросить:
□ ID партии или описание (дата, культура, зона)
□ Текущий статус в Sheets (лист Агрономия)
□ Результаты QA записи

Статусы партии:
  production   — в производстве
  harvest      — собрана, ожидает упаковки
  packed       — упакована, ожидает отгрузки
  on_hold      — удержана, требует проверки
  released     — выпущена, разрешена к отгрузке
  dispatched   — отгружена клиенту
  recalled     — отозвана
```

**Чеклист перед отгрузкой:**

```
□ Партия имеет harvest_date и batch_id
□ QA осмотр проведён (записан в Агрономия)
□ Температура хранения в норме (2–8°C для огурца)
□ Упаковка целая, маркировка читается
□ Вес/количество соответствует заказу
□ Срок годности позволяет доставку + 2 дня
□ Клиент подтверждён в заказе
```

## Режим 2 — Поставить на Hold

```
Триггеры для автоматического Hold:
• Визуальные дефекты > 10% продукции в партии
• Отклонение температуры хранения > 4 часов
• Жалоба клиента на предыдущую партию той же зоны
• Нарушение санитарии в зоне сбора
• Неизвестная болезнь или вредитель на растениях-донорах
• Отсутствие QA записи для партии

Алгоритм Hold:
1. Записать алерт в Sheets "Алерты" (критичность 4)
2. Зафиксировать причину Hold в Sheets "Агрономия" (событие = "Hold")
3. Уведомить: "Партия [X] поставлена на Hold. Причина: [причина]. Требуется проверка."
4. Создать задачу в "Задачи": "Проверить партию [X] — Hold" | Приоритет: 5 | Дедлайн: сегодня
```

**Запись в "Алерты":**

```
[дата-время] | Food Safety | [зона/партия] | Hold: [причина] | 4 | открыт | qa | [план действий]
```

## Режим 3 — Выпуск партии ⚠️

> **КРИТИЧНО:** Выпуск партии требует явного подтверждения владельца.

```
Алгоритм выпуска:
1. Показать владельцу сводку:
   - Партия: [ID]
   - Hold с: [дата]
   - Причина Hold: [причина]
   - Результаты проверки: [что проверено, что нашли]
   - Рекомендация QA: [выпустить / утилизировать / частичный выпуск]

2. Запросить подтверждение:
   ⚠️ ТРЕБУЕТСЯ ОДОБРЕНИЕ
   Действие: выпуск партии [ID] — [X кг] [культура]
   QA проверка: [результат]
   → Подтверждаете выпуск? ("да" / "нет" / "частично")

3. После подтверждения:
   - Записать в Агрономия: событие = "Released", примечание = "QA approved by owner"
   - Закрыть алерт Hold в "Алерты"
   - Обновить статус задачи
```

## Режим 4 — HACCP / CCP мониторинг

**Критические контрольные точки (CCP) для Pepino Pick:**

| CCP   | Параметр                           | Норма                    | Отклонение = Hold       |
| ----- | ---------------------------------- | ------------------------ | ----------------------- |
| CCP-1 | Температура хранения (холодильник) | 2–8°C                    | < 0°C или > 10°C > 2ч   |
| CCP-2 | Температура транспортировки        | < 10°C                   | > 12°C > 1ч             |
| CCP-3 | Санитарная обработка инструмента   | Перед каждым срезом      | Пропуск > 2 циклов      |
| CCP-4 | Вода для мойки продукции           | Питьевая / фильтрованная | Неизвестный источник    |
| CCP-5 | Личная гигиена персонала           | Без болезней, перчатки   | Нарушение = отстранение |

**При отклонении CCP:**

```
1. Немедленно зафиксировать в "Алерты" (критичность 5)
2. Остановить затронутую партию (перевести на Hold)
3. Определить объём затронутой продукции
4. Уведомить владельца
5. Создать задачу на корректирующее действие (CAPA)
```

## Режим 6 — Инцидент качества (жалоба)

```
При получении жалобы на качество:

1. ЗАФИКСИРОВАТЬ в "Алерты":
   [дата] | Food Safety | [клиент] | Жалоба: [описание] | [критичность] | открыт | qa | [план]

2. УСТАНОВИТЬ СВЯЗЬ:
   → Найти партию по дате поставки и культуре
   → Проверить: есть ли другие клиенты из той же партии?

3. ОЦЕНИТЬ МАСШТАБ:
   □ Единичный случай → мониторинг
   □ Повторный / несколько клиентов → Hold всей партии
   □ Подозрение на food safety risk → немедленно уведомить владельца + Hold

4. ПЛАН ОТВЕТА КЛИЕНТУ:
   → Шаблон извинения и замены (через greenhouse-sales skill)
   → Сроки ответа: 24ч для tier-1 клиентов (рестораны)

5. CAPA (Corrective Action):
   → Задача в "Задачи": найти корень проблемы + устранить
```

## Режим 7 — Аудит / Готовность к проверке

**Быстрый чеклист готовности:**

```
ДОКУМЕНТАЦИЯ:
□ Журналы урожая заполнены (лист Агрономия)
□ Все партии имеют batch_id и QA запись
□ Hold-записи закрыты или имеют план
□ Алерты критичности 4-5 проработаны

ОПЕРАЦИИ:
□ Холодильник работает, температура в норме
□ Инструменты продезинфицированы
□ Личные карточки сотрудников актуальны
□ Зоны промаркированы

AI СИСТЕМА:
□ Последние агрономические записи < 48ч
□ Нет открытых алертов > 5 дней
□ Решения задокументированы в "Решения"
```

## Запись данных в Google Sheets

**В лист "Алерты"** (стандарт QA):

```js
await appendToSheet(
  OS_ID,
  [
    [
      new Date().toISOString(), // дата-время
      "Food Safety", // тип
      "Партия/Зона", // источник
      "Описание инцидента", // описание
      4, // критичность 1-5
      "открыт", // статус
      "qa", // ответственный
      "План реакции", // реакция
    ],
  ],
  "Алерты",
);
```

**В лист "Задачи"** (CAPA):

```js
await appendToSheet(
  OS_ID,
  [
    [
      new Date().toISOString().split("T")[0], // дата создания
      "CAPA: [описание проблемы]", // задача
      "QA", // отдел
      "qa_agent", // ответственный
      5, // приоритет (1=низкий, 5=критичный)
      "YYYY-MM-DD", // дедлайн (через 48ч)
      "открыта", // статус
      "", // дата закрытия
      "Linked to: [партия/алерт]", // примечание
    ],
  ],
  "Задачи",
);
```

## Ограничения

- Skill не выпускает партии самостоятельно — только с подтверждением владельца
- При любом подозрении на food safety risk — немедленный Hold, без исключений
- Финальное решение по отзыву продукта — только владелец
