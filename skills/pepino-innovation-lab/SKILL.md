## Режим 2 — Waste-to-Value

**Триггеры:** "лишний урожай", "нестандарт", "отходы", "не продать свежим", "избыток", "что делать с лишним"

### Алгоритм Waste-to-Value

```
ШАГ 1: ОЦЕНИТЬ ОБЪЁМ И ПРИЧИНУ
  → Сколько кг? Какая культура? Почему не продаётся?
  → Варианты: избыток сезонный / качество ниже стандарта / клиент отменил заказ

ШАГ 2: MATRIX РЕШЕНИЙ (ROI-based)

  Продукт           ROI ориентир   Срок   Сложность
  ─────────────────────────────────────────────────
  Ферментация:      3–5x себест.   7–30д  Средняя
  Соус/маринад:     4–7x себест.   1–3д   Низкая
  Сушка:            5–10x себест.  3–5д   Средняя
  Компостирование:  0 (но экономит) —     Низкая
  Снизить цену:     0.5–1x         сегодня Нулевая

ШАГ 3: РЕКОМЕНДОВАТЬ ЛУЧШИЙ ПУТЬ
  → На основе объёма, срока годности, имеющегося оборудования

ШАГ 4: ОТКРЫТЬ КЕЙС (если ферментация/переработка > 5 кг)
  → case_type: waste_conversion
  → Связать с fermentation workflow

ШАГ 5: ЗАЛОГИРОВАТЬ RECOVERY ROI
  → Записать в Sheets: исходный объём → конечный продукт → выручка → ROI
```

### Шаблон Waste Recovery

```
♻️ WASTE-TO-VALUE АНАЛИЗ
━━━━━━━━━━━━━━━━━━━━━━━━
Продукт:       [культура]
Объём:         [X кг]
Причина:       [избыток / нестандарт / возврат]
Срок годности: [N дней осталось]

Варианты конвертации:
  1. [опция 1]: ROI ~[X]x | Требует: [что нужно] | Срок: [N дней]
  2. [опция 2]: ROI ~[X]x | Требует: [что нужно] | Срок: [N дней]
  3. Снижение цены: продать за [X] ARS → Recovery [Y]%

Рекомендация: [опция] — [1-2 предложения обоснования]

⚠️ Требует одобрения если запускаем ферментацию > 20 кг
━━━━━━━━━━━━━━━━━━━━━━━━
```

### Chef Intelligence Pipeline

```
PIPELINE: Шеф-повар → Продажа
─────────────────────────────
Этап 1: Первый контакт / дегустация
  → Записать шефа в Sheets "Маркетинг" (тип = "Chef Contact")
  → Зафиксировать ресторан, кухня, бюджет, текущие поставщики

Этап 2: Пилот-поставка (1-2 кг)
  → Batch с пометкой "Chef Trial"
  → Запросить обратную связь через 48ч

Этап 3: Sensory feedback
  → Записать сенсорный профиль (см. выше)
  → Обновить "Chef Preferences" в профиле клиента

Этап 4: Коммерческое предложение
  → greenhouse-sales: сгенерировать КП с историей продукта
  → Учесть chef preferences в позиционировании
```
