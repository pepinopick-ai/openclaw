# Pepino Pick Agent OS v2 -- Policy Engine

> Формальный движок политик, определяющий границы автономии каждого агента.
> 5 уровней действий: от READ-ONLY до HUMAN-ONLY.

**Версия:** 2.0
**Дата:** 2026-03-21
**Валюта бюджетных порогов:** ARS (аргентинское песо)

---

## Принципы

1. **Principle of Least Authority** -- агент получает минимально необходимый уровень автономии для своей задачи.
2. **Fail-Safe Default** -- при неопределённости уровня действие классифицируется на уровень ВЫШЕ (более строгий).
3. **Audit Everything** -- каждое действие уровня L1+ записывается в Google Sheets с timestamp, agent_id, case_id.
4. **Human in the Loop** -- уровни L2+ всегда информируют CEO, уровни L3+ блокируются до одобрения.
5. **No Silent Failures** -- если агент не может выполнить действие, он создаёт алерт, а не молчит.

## Level 0: READ-ONLY

**Описание:** Любой агент может читать любые данные без ограничений и одобрений.

### Действия уровня L0

- Чтение любого листа Google Sheets (GET)
- Чтение Obsidian vault (СОПы, мемо, решения)
- Запрос курсов валют (dolarapi.com)
- Чтение KPI, метрик, дашбордов
- Проверка статуса партии/батча
- Проверка остатков склада
- Просмотр pipeline продаж
- Просмотр реестра рисков
- Поиск по базе знаний
- Геокодирование и маршрутизация (maps)
- Парсинг публичных сайтов (scrape)
- Запрос диагностики фото (без записи результата)

### Правила L0

| Параметр         | Значение                      |
| ---------------- | ----------------------------- |
| Агенты           | Все 31                        |
| Одобрение        | Не требуется                  |
| Документация     | Не требуется                  |
| Audit trail      | Нет (read-only не логируется) |
| Бюджетный импакт | 0 ARS                         |

## Level 1: AUTO-EXECUTE (низкий риск)

**Описание:** Агент выполняет действие автоматически. CEO информируется постфактум через дайджест или по запросу.

**Бюджетный порог:** < 50,000 ARS прямого воздействия.

- Создание алерта (Алерты) любого уровня
- Обновление статуса задачи (Задачи)
- Логирование кейса (Кейсы) через dispatcher
- Отправка информационного Telegram-сообщения CEO
- Запись замеров ферментации (pH, соль, температура)
- Перевод статуса батча (active -> monitoring -> tasting)
- Обновление курсов валют в листе "Курсы валют"
- Создание карточки поставщика/шефа/клиента
- Обновление pipeline стадии (lead -> contacted)
- Создание follow-up задачи
- Запись расхода субстрата/расходников
- Фиксация решения встречи в листе "Решения"
- Обновление карточки договора (статус, даты)
- Запись R&D эксперимента (open)
- Рассчитать и записать unit economics
- Обновить KPI-метрики

## Level 2: PROPOSE-ONLY (средний риск)

**Описание:** Агент формирует рекомендацию и отправляет CEO на одобрение через Telegram. Действие НЕ выполняется до получения явного подтверждения.

**Бюджетный порог:** 50,000 -- 250,000 ARS прямого воздействия.

### Действия уровня L2

- Изменение цены продукта (рекомендация)
- Добавление нового поставщика (без истории)
- Маркетинговый контент для публикации (черновик)
- Назначение задачи сотруднику (с дедлайном и приоритетом)
- Изменение стадии клиента (at_risk -> churned)
- Заказ материалов 50K-250K ARS
- Скидка клиенту >5%
- Рекомендация Kill/Iterate по продукту
- Предложение waste-to-value обработки
- Назначение дегустации tier-2+ шефу
- Рекомендация изменения бюджета категории
- Предложение pilot batch нового SKU
- Рекомендация смены модели AI (cost governance)
- Предложение нового OKR
- Рекомендация маршрута доставки с изменением логистики

## Level 3: DRAFT-WITH-APPROVAL (высокий риск)

**Описание:** Агент готовит полный пакет документов (драфт, расчёты, обоснование), но НИЧЕГО не отправляет и не подписывает. CEO получает пакет, ревьюит и принимает решение.

**Бюджетный порог:** 250,000 -- 1,500,000 ARS прямого воздействия.

### Действия уровня L3

- Контракт с клиентом/поставщиком (драфт)
- Решение о выпуске партии (release) -- QA пишет рекомендацию
- CAPEX-проект до 1.5M ARS (полный пакет NPV/IRR/payback)
- План запуска нового SKU (GTM + финмодель)
- Изменение product mix (убрать/добавить линейку)
- Заказ оборудования 250K-1.5M ARS
- Предоставление эксклюзивных условий tier-1 клиенту
- Найм сотрудника (карточка + бюджет)
- Подача заявки на госпрограмму (FONDEP, BNA)
- Активация contingency plan по критическому риску
- Изменение системы ирригации/освещения
- Инвесторская презентация (pitch deck draft)
- Bulk pricing для крупного клиента (>6 месяцев)

### Правила L3

| Параметр         | Значение                                                                                                                                                             |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Агенты           | legal, qa-food-safety, capital, product-manager, procurement, team-ops, innovation-lab, risk, greenhouse-tech                                                        |
| Одобрение        | CEO -- явное "APPROVE" через Telegram или голосом                                                                                                                    |

| Timeout          | 4 часа для срочных (release партии), 72 часа для остальных                                                                                                           |

## Level 4: HUMAN-ONLY (критический)

**Описание:** Агент НЕ МОЖЕТ инициировать, выполнить или даже сформировать драфт. Агент может только собрать информацию и передать CEO. Все действия выполняются человеком лично.

**Бюджетный порог:** > 1,500,000 ARS или стратегический характер.

### Действия уровня L4

- Найм/увольнение сотрудников (решение)
- Юридические претензии и иски >500K ARS
- Контракты >12 месяцев
- CAPEX >7,500,000 ARS (Board level)
- Стратегический pivot бизнеса
- Публикация брендированного контента (финальная)
- Подписание договора аренды/покупки земли
- Контакт с регулятором (SENASA, ANMAT) от имени компании
- Оффер инвестору / принятие инвестиций
- Изменение юридической формы бизнеса
- Публичные PR-заявления от имени Pepino Pick
- Отзыв продукции с рынка
- Смена банка / открытие нового счёта
- Страховые решения
- Решение о закрытии линейки производства

### Правила L4

| Параметр         | Значение                                                                                                                  |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------- |
| Агенты           | Ни один агент не может выполнить. Могут информировать: legal, capital, risk, team-ops, brand, shadow-ceo                  |
| Одобрение        | CEO лично (не через бота)                                                                                                 |
| Документация     | Агент может подготовить briefing package, но решение и действие -- только CEO                                             |
| Approval flow    | Агент собирает данные -> briefing package -> Telegram CEO "REQUIRES YOUR PERSONAL ACTION" -> CEO действует самостоятельно |
| Timeout          | Нет (CEO принимает когда готов)                                                                                           |
| Escalation       | pepino-shadow-ceo напоминает через 7 дней, далее weekly reminder                                                          |
| Audit trail      | case_id, briefing package, дата передачи CEO, результат (после факта, записывает CEO или по его указанию)                 |
| Бюджетный импакт | > 1,500,000 ARS или любой стратегический                                                                                  |

### Формат briefing (Telegram)

```
HUMAN ACTION REQUIRED [L4]
Case: [case_id]
Domain: [legal / capital / strategic / HR]
Summary: [2-3 предложения]
Briefing: [ссылка на полный пакет]
Why human-only: [причина]
No timeout -- take your time.
```

## Escalation Protocol

### Путь эскалации по уровням

```
L1 (auto) -- нет эскалации (логируется)
     |
L2 (propose) -- CEO не ответил 24h
     |
     v
pepino-shadow-ceo -- напоминание CEO
     |
     48h без ответа
     |
     v
Blocked Decision -- фиксация в листе Решения (status: blocked)
     |
     weekly-review подсвечивает
     |
     v
CEO разблокирует вручную
```

### Путь эскалации по критичности

```
Severity 1-2 (low):  L1 auto -> лог
Severity 3 (medium): L1 auto -> алерт Telegram
Severity 4 (high):   L2/L3 -> CEO approval
Severity 5 (critical): L3/L4 -> CEO немедленно + shadow-ceo
```

## Конфликт уровней

Если действие попадает между двумя уровнями:

1. **Бюджет решает** -- если сумма чётко определена, уровень определяется по порогу.
2. **При неопределённости -- уровень ВЫШЕ** -- лучше запросить лишнее одобрение, чем пропустить.
3. **Комбинированные действия** -- уровень определяется по САМОМУ высокому компоненту.
4. **Повторяющиеся действия** -- суммируются за неделю. Если кумулятивный импакт >порога текущего уровня, следующее действие переходит на уровень выше.

### Ежемесячный аудит

pepino-controller + pepino-shadow-ceo проводят ежемесячный аудит:

1. Все L2+ действия проверяются на наличие одобрения.
2. Все L1 действия проверяются на корректность бюджетного импакта.
3. Отчёт аудита записывается в лист "Решения" с case_id = AUDIT-YYYYMM.
4. Нарушения (действие без одобрения, превышение бюджетного порога) -- эскалируются CEO как severity 5 алерт.
