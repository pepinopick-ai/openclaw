# pepino-risk

Управление рисками Pepino Pick: реестр рисков, мониторинг, митигация, кризисные сценарии, early warning indicators.

## Триггеры

риск, угроза, кризис, уязвимость, сценарий, что если, форс-мажор, contingency, plan B, риски бизнеса, страхование, EWI, early warning, мониторинг рисков, риск-матрица, катастрофа, потеря клиента, потеря поставщика, болезнь урожая, финансовый риск, операционный риск

## Режимы работы

### 1. РЕЕСТР РИСКОВ

**Категории рисков:**

🌱 **Агрономические:**

- Болезнь / контаминация культуры
- Отказ системы полива / климат-контроля
- Некачественный субстрат / мицелий
- Аномальная жара / холод

💰 **Финансовые:**

- Резкая инфляция → потеря маржи
- Потеря крупного клиента (>20% выручки)
- Неоплата дебиторки (>60 дней)
- Курсовой риск ARS/USD
- Кассовый разрыв

🚚 **Операционные:**

- Потеря ключевого поставщика
- Сбой доставки
- Потеря ключевого сотрудника
- Отключение электричества >4h
- Поломка критического оборудования

🏛️ **Регуляторные:**

- Изменение норм SENASA/ANMAT
- Налоговые изменения
- Проверка / санкция

🎯 **Стратегические:**

- Появление сильного конкурента
- Потеря рыночной ниши
- Технологическое устаревание

**Карточка риска:**

```
ID: RISK-[YYYYMMDD]-[категория]-[N]
Название: [краткое описание]
Категория: [agronomy / financial / operational / regulatory / strategic]
Вероятность: [1=низкая / 2=средняя / 3=высокая]
Влияние: [1=незначительное / 2=умеренное / 3=критическое]
Рейтинг: [вероятность × влияние = 1-9]
Статус: [active / mitigated / monitoring / materialized]
Владелец: [кто отвечает за митигацию]
Mitigation: [что делаем для снижения риска]
Contingency: [что делаем если риск реализовался]
Early Warning: [что наблюдаем как ранний сигнал]
Обновлено: [дата]
```

**Риск-матрица (текущие топ-риски):**

```
         │ Влияние 1 │ Влияние 2 │ Влияние 3
─────────┼───────────┼───────────┼───────────
Вер. 3   │     3     │     6     │    🔴 9
Вер. 2   │     2     │    🟡 4  │    🔴 6
Вер. 1   │     1     │     2     │    🟡 3
```

🔴 Рейтинг 6-9: требует немедленной митигации
🟡 Рейтинг 3-5: мониторинг и план митигации
⚪ Рейтинг 1-2: принять и мониторить

### 2. EARLY WARNING INDICATORS (EWI)

Метрики раннего предупреждения — мониторировать еженедельно:

**Агрономические EWI:**

```
🟡 Алерт если:
  - Урожайность вешенки <2.5 кг/блок за цикл (норма 3-4)
  - Потери от брака >8% от урожая (норма <5%)
  - pH субстрата отклонение >0.5 от нормы 2 дня подряд
  - Влажность в теплице <70% или >95% более 6 часов

🔴 Алерт если:
  - Визуальные признаки контаминации на >5% блоков
  - Полный отказ климат-контроля
  - Урожай ниже 50% нормы 2 недели подряд
```

**Финансовые EWI:**

```
🟡 Алерт если:
  - Маржа по SKU падает ниже 25%
  - Дебиторка >45 дней у клиента Tier 1
  - Cash balance <500,000 ARS
  - Один клиент = >30% выручки

🔴 Алерт если:
  - Маржа <15% (ниже операционной безубыточности)
  - Cash balance <200,000 ARS
  - Клиент Tier 1 отменил заказ без объяснений
  - Поставщик отказал в поставке без предупреждения
```

**Операционные EWI:**

```
🟡 Алерт если:
  - Ключевой сотрудник болен >3 дней
  - Поставщик задержал поставку >2 дней
  - 1 поставщик = >50% объёма любого критичного ресурса

🔴 Алерт если:
  - Потеря ключевого сотрудника (увольнение)
  - Полная невозможность поставки критичного ресурса
  - Авария в теплице (затопление, пожар, etc)
```

### 3. КРИЗИСНЫЕ СЦЕНАРИИ И PLAYBOOKS

**Сценарий A: Контаминация партии**

```
Trigger: визуальная плесень / нетипичный запах / положительный тест
Действия:
  1. СТОП реализации этой партии немедленно
  2. Открыть case pepino-qa-food-safety: HOLD всей партии
  3. Трассировка: какой субстрат, мицелий, условия
  4. Уведомить клиентов получивших → возврат/замена
  5. Root cause analysis в течение 24h
  6. Записать в pepino-knowledge: урок + постмортем
Ответственный: CEO + агроном
```

**Сценарий B: Потеря крупного клиента**

```
Trigger: клиент Tier 1 отказался или перестал покупать
Действия:
  1. Звонок CEO клиенту в течение 24h — выяснить причину
  2. Оценить gap в выручке (сколько ARS/мес теряем)
  3. Активировать pepino-sales-crm: ускорить 2-3 prospect из pipeline
  4. Временно сократить объём производства проблемного SKU
  5. Analysys причин — продукт / цена / сервис / конкурент
Ответственный: CEO + sales
```

**Сценарий C: Кассовый разрыв**

```
Trigger: cash < 300,000 ARS или прогноз дефицита в течение 4 недель
Действия:
  1. pepino-finance-tools: точный cash flow прогноз на 30 дней
  2. Приоритет инкассации: позвонить всем с >30 дней дебиторки
  3. Оплата только критичных расходов (зарплата, аренда, электричество)
  4. Рассмотреть предоплатный дискаунт клиентам (3-5%)
  5. Если дефицит >1M ARS → активировать pepino-capital
Ответственный: CEO + controller
```

**Сценарий D: Потеря ключевого поставщика**

```
Trigger: основной поставщик субстрата / мицелия недоступен
Действия:
  1. pepino-procurement: активировать резервного поставщика
  2. Оценить запас на N дней производства
  3. Если запаса <7 дней → срочный поиск альтернативы
  4. Уведомить клиентов о возможных задержках
  5. Зафиксировать урок: нельзя иметь <2 поставщиков критичных ресурсов
Ответственный: CEO + закупки
```

### 4. РИСК-АУДИТ (ЕЖЕКВАРТАЛЬНЫЙ)

**Чеклист ежеквартального аудита:**

```
Агрономические риски:
  ☐ Проверен реестр болезней и паразитов за квартал
  ☐ Оборудование климат-контроля прошло ТО
  ☐ Есть резервный запас субстрата и мицелия на 2 недели
  ☐ Задокументированы SOP на случай контаминации

Финансовые риски:
  ☐ Проанализирована концентрация клиентов (нет клиента >40% выручки)
  ☐ Cash reserve достаточен для 6 недель операций
  ☐ Хеджирование инфляции: цены пересмотрены <3 месяцев назад
  ☐ Дебиторка >60 дней = 0 или активно взыскивается

Операционные риски:
  ☐ Все ключевые процессы задокументированы (нет bus factor = 1)
  ☐ Для каждого критичного поставщика есть альтернатива
  ☐ Страховка актуальна (имущество, ответственность)

Стратегические риски:
  ☐ Мониторинг конкурентов проведён
  ☐ Customer satisfaction проверен (тастинги, отзывы)
```
