# 🏆 Pepino Pick — Profit Engine

Система управления рентабельностью продуктов: трекинг маржи/м², классификация Winner/Watch/Loser, запуск и оценка экспериментов по модели Kill/Iterate/Scale, еженедельные отчёты с actionable-решениями.

**API Gateway:** `http://localhost:4000`

**Источники данных:**

| Endpoint              | Что содержит                                                                                                         |
| --------------------- | -------------------------------------------------------------------------------------------------------------------- |
| GET /products         | product, profit_per_m2_usd, margin_pct, area_m2, annual_profit_usd, priority, cycles_per_year, yield_kg_per_m2_cycle |
| GET /sales            | Дата, Клиент, Продукт, Кол-во кг, Цена ARS/кг, Сумма ARS, Сумма USD, Статус                                          |
| GET /production       | Дата, Продукт, Урожай кг, Отход кг, % отхода                                                                         |
| GET /expenses         | Дата, Наименование, Сумма ARS, Сумма USD                                                                             |
| GET /kpi              | Агрегированные KPI                                                                                                   |
| GET /pnl              | P&L данные                                                                                                           |
| POST /log/experiments | Запись экспериментов                                                                                                 |

## 5 режимов работы

### Режим 1 — Product Profitability Dashboard (Дашборд рентабельности)

Когда пользователь спрашивает о рентабельности, марже, лучших/худших продуктах.

**Шаги:**

```
ШАГ 1: Получить данные
  → GET /products — базовые параметры продуктов (area_m2, cycles_per_year, yield_kg_per_m2_cycle)
  → GET /sales — фактические продажи за последние 2 месяца (фильтр по Дата)
  → GET /production — фактическое производство за последние 2 месяца
  → GET /expenses — расходы за последние 2 месяца

ШАГ 2: Рассчитать метрики по каждому продукту
  → revenue_per_m2_month = SUM(Сумма USD за месяц по продукту) / area_m2
  → cost_per_m2_month = пропорциональная доля расходов по площади:
      total_expenses_usd * (area_m2_продукта / total_area_m2) / кол-во_месяцев
  → gross_margin_per_m2 = revenue_per_m2_month − cost_per_m2_month
  → margin_pct = gross_margin_per_m2 / revenue_per_m2_month * 100
  → contribution_margin = revenue_per_m2_month − variable_cost_per_m2_month
  → waste_pct = SUM(Отход кг) / SUM(Урожай кг) * 100  (из /production)
  → trend = сравнить margin_pct текущего месяца vs предыдущего:
      delta > +3pp → "improving"
      delta < -3pp → "declining"
      иначе → "stable"

ШАГ 3: Классифицировать продукт
  → 🏆 Winner: margin_pct > 50% AND trend != "declining"
  → ⚠️ Watch: (margin_pct 30–50%) OR (trend == "declining" AND margin_pct > 30%)
  → 🔴 Loser: margin_pct < 30% OR waste_pct > 15%

ШАГ 4: Сформировать рекомендации
  → Winner → "Масштабировать: увеличить area_m2, оптимизировать цикл"
  → Watch → "Мониторить: проверить причину снижения, рассмотреть ценообразование"
  → Loser → "Решение: Kill (убрать) или Iterate (изменить процесс/цену)"
```

### Режим 3 — Kill/Iterate/Scale Review (Обзор экспериментов)

Когда пользователь запрашивает обзор эксперимента, просит принять решение kill/iterate/scale.

**Шаги:**

```
ШАГ 1: Получить данные эксперимента
  → Из Sheets: experiment_id, baseline, targets, status, decision_deadline
  → Из /sales, /production: актуальные метрики за период эксперимента

ШАГ 2: Рассчитать delta
  → delta_margin = actual_margin − baseline_margin
  → delta_revenue_m2 = actual_revenue_m2 − baseline_revenue_m2
  → delta_waste = actual_waste − baseline_waste
  → target_achievement_pct = (actual − baseline) / (target − baseline) * 100

ШАГ 3: Определить тренд
  → Разбить период на недели, посчитать метрики по неделям
  → Линейная аппроксимация: slope > 0 → improving, slope < 0 → declining

ШАГ 4: Рекомендация
  → KILL:
      delta_margin < 0 AND тренд declining AND прошло > 50% срока
      ИЛИ сработало stop_condition
  → ITERATE:
      delta_margin > 0 BUT target_achievement < 70%
      ИЛИ тренд improving BUT ещё рано для выводов (< 50% срока)
  → SCALE:
      target_achievement >= 100% AND тренд stable/improving
      ИЛИ сработало scale_condition

ШАГ 5: Сформировать Decision Memo
```

### Режим 5 — Margin per m2 Trend (Тренд маржи/м²)

Анализ временных рядов маржи по продуктам для Grafana и алертов.

**Шаги:**

```
ШАГ 1: Получить исторические данные
  → GET /sales — за последние 6 месяцев
  → GET /production — за последние 6 месяцев
  → GET /expenses — за последние 6 месяцев
  → GET /products — текущие area_m2 по продуктам

ШАГ 2: Агрегировать помесячно
  → Для каждого продукта и каждого месяца:
    monthly_revenue_m2 = SUM(Сумма USD) / area_m2
    monthly_cost_m2 = proportional_expenses / area_m2
    monthly_margin_m2 = monthly_revenue_m2 − monthly_cost_m2
    monthly_margin_pct = monthly_margin_m2 / monthly_revenue_m2 * 100

ШАГ 3: Рассчитать тренды
  → Для каждого продукта: линейная регрессия margin_m2 по месяцам
  → slope > 0 → "growing", slope < 0 → "declining", |slope| < threshold → "stable"
  → R² для оценки достоверности тренда

ШАГ 4: Алерты
  → Если margin_m2 текущего месяца < margin_m2 предыдущего на > 10%:
    АЛЕРТ: "Маржа/м² [продукт] упала на XX% (с $X до $Y)"
  → Записать алерт в Sheets лист "Алерты":
    [дата] | Margin Drop | pepino-profit-engine | [описание] | [критичность] | открыт | pepino-shadow-ceo | [рекомендация]

ШАГ 5: Grafana-ready данные
  → Вывести таблицу в формате, пригодном для визуализации

### Режим 6 — Competitive Pricing Analysis (Конкурентный анализ цен)

**Шаги:**

```
ШАГ 1: Получить наши цены
  → GET /products — текущие цены продуктов Pepino Pick (price_ars_kg, margin_pct)
  → Продукты для анализа: вешенка, микрозелень, огурец, съедобные цветы

ШАГ 2: Получить рыночные данные
  → Вариант A (автоматический): запустить скрипт
    node skills/pepino-google-sheets/auto-pricing.cjs --dry-run
  → Вариант B (ручной): использовать mercadolibre-analyst для поиска по запросам:
    - вешенка: "hongos gírgolas frescas", "gírgolas frescas kg"
    - микрозелень: "microgreens frescos", "microverdes bandeja"
    - огурец: "pepino fresco kg", "pepino holandes invernadero"

  → Для каждого продукта рассчитать:

## Правила Profit Engine (Guardrails)

```
ПРАВИЛО 1: НИКОГДА не выдавать рекомендацию KILL без baseline данных минимум за 4 недели.
           При недостатке данных: "Недостаточно данных для решения. Требуется ещё X недель."

ПРАВИЛО 2: НИКОГДА не использовать плановые данные как фактические.
           Только реальные записи из /sales, /production, /expenses.

ПРАВИЛО 3: НИКОГДА не рекомендовать SCALE без проверки:
           → Есть ли свободная площадь?
           → Хватит ли ресурсов (субстрат, рабочие руки)?
           → Есть ли рынок сбыта (demand oracle)?

ПРАВИЛО 4: Все суммы в USD. ARS суммы конвертировать по актуальному Blue Dollar.
           Указывать курс, использованный для конвертации.

ПРАВИЛО 5: Waste% > 15% — ВСЕГДА помечать как проблему, даже если маржа высокая.
           Высокий отход = скрытый убыток + операционная проблема.

ПРАВИЛО 6: Trend analysis требует минимум 2 полных месяца данных.
           При 1 месяце: пометить "trend: insufficient data".

ПРАВИЛО 7: SCALE решение с увеличением area_m2 > 20% или CAPEX > 1,000,000 ARS
           → APPROVAL_REQUIRED (использовать стандартный STOP-блок диспетчера).

ПРАВИЛО 8: Эксперимент без decision_deadline — недопустим.
           Максимальный срок эксперимента: 90 дней. Минимальный: 14 дней.

ПРАВИЛО 9: При расчёте cost_per_m2 — использовать пропорциональное распределение
           расходов по площади (area_m2). Не делить расходы поровну.

ПРАВИЛО 10: Не рекомендовать одновременно KILL для > 2 продуктов.
            Массовый KILL = стратегическое решение → pepino-shadow-ceo.
```

## LLM Tier Routing

Profit Engine использует разные tier-ы в зависимости от режима работы:

| Режим                         | Tier       | Обоснование                                                |
| ----------------------------- | ---------- | ---------------------------------------------------------- |
| Режим 1 (Dashboard)           | T2 Medium  | Агрегация данных, форматирование — DeepSeek V3 / Kimi K2.5 |
| Режим 2 (Experiment Create)   | T2 Medium  | Расчёт baseline, запись — стандартная сложность            |
| Режим 3 (Kill/Iterate/Scale)  | T3 Complex | Стратегическое решение, требует reasoning — Claude Sonnet  |
| Режим 4 (Weekly Report)       | T2 Medium  | Агрегация + форматирование                                 |
| Режим 5 (Margin Trend)        | T3 Complex | Статистический анализ, линейная регрессия, алерты          |
| Режим 6 (Competitive Pricing) | T2 Medium  | Сбор данных + сравнение, без стратегических решений        |

Классификация через: `node /home/roman/openclaw/skills/pepino-google-sheets/llm-router.cjs classify --skill pepino-profit-engine --intent <режим>`
