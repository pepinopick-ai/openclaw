# 📊 Pepino Pick — Еженедельный Executive Review

Агрегирует все операционные данные за неделю в единый дайджест для владельца.

**Google Sheets ID:** `1AB9nkHfCu8_12dwn72tWKZOiqHAfAhI3AHOjeAAByoc`

---

## Алгоритм (запускать каждый понедельник или по запросу)

```
ШАГ 1: ОПРЕДЕЛИТЬ ПЕРИОД
  → По умолчанию: прошлая неделя (Пн–Вс)
  → Если указан период — использовать его

ШАГ 2: СОБРАТЬ ДАННЫЕ из всех листов (параллельно)
  → Агрономия: урожай, события, проблемы
  → Финансы: приход/расход, итого за неделю
  → Маркетинг: продажи, клиенты, каналы
  → Логистика: отгрузки, маршруты, инциденты
  → Задачи: закрыто/открыто/просрочено
  → Алерты: критичность, статусы
  → Закупки: расходы, новые поставщики
  → Решения: принятые решения за неделю

ШАГ 3: РАССЧИТАТЬ KPI

ШАГ 4: ОПРЕДЕЛИТЬ TOP-3 ПРОБЛЕМЫ и TOP-3 УСПЕХИ

ШАГ 5: СФОРМИРОВАТЬ ДАЙДЖЕСТ (см. шаблон)

ШАГ 6: ЗАПИСАТЬ в лист "Итоги дня" (запись "Итоги недели")

ШАГ 7: ВЫВЕСТИ владельцу
```

---

## Данные для сбора по листам

### Лист "Агрономия"

Фильтр: строки за отчётный период.
Считать:

- Суммарный урожай (кг) по культурам
- Количество событий типа "инокуляция" / "срез" / "посев"
- Количество проблем (записи с ключевыми словами: болезнь, вредитель, отклонение, проблема)
- EC/pH отклонения если зафиксированы

### Лист "Финансы"

Фильтр: строки за период.
Считать:

- Общий приход (тип=приход/продажа) в ARS
- Общий расход в ARS по категориям
- Нетто за неделю (приход - расход)
- Топ-3 статьи расходов

### Лист "Маркетинг"

Фильтр: строки за период.
Считать:

- Количество продаж
- Суммарная выручка ARS
- Уникальные клиенты
- Разбивка по каналам (прямые / рестораны / другие)

### Лист "Логистика"

Фильтр: строки за период.
Считать:

- Количество отгрузок
- Суммарный отгруженный объём (кг)
- Инциденты доставки
- Возвраты

### Лист "Задачи"

Фильтр: строки созданные ИЛИ закрытые за период.
Считать:

- Закрыто задач
- Открыто новых задач
- Просрочено (статус != закрыта И дедлайн < сегодня)
- Задачи по отделам

### Лист "Алерты"

Фильтр: строки за период.
Считать:

- Всего алертов
- По критичности (1-5)
- Нерешённые алерты (статус=открыт)
- Топ алертов недели

### Лист "Закупки"

Фильтр: строки за период.
Считать:

- Суммарные закупки ARS/USD
- Количество поставщиков
- Новые позиции

## Как читать данные из Google Sheets

Используй MCP tool `mcp__googlesheets__get_sheet_data` или `pepino-google-sheets` skill:

```js
// Через pepino-google-sheets skill:
const OS_ID = "1AB9nkHfCu8_12dwn72tWKZOiqHAfAhI3AHOjeAAByoc";

// Читать листы параллельно:
const [agro, finance, marketing, logistics, tasks, alerts, purchases] = await Promise.all([
  readSheet(OS_ID, "Агрономия"),
  readSheet(OS_ID, "Финансы"),
  readSheet(OS_ID, "Маркетинг"),
  readSheet(OS_ID, "Логистика"),
  readSheet(OS_ID, "Задачи"),
  readSheet(OS_ID, "Алерты"),
  readSheet(OS_ID, "Закупки"),
]);
```

Фильтрация по дате: колонка A в каждом листе — дата в формате YYYY-MM-DD.

## Запись итогов в "Итоги дня"

После формирования дайджеста записать строку:

```
Колонки: Дата | Агрономия | Логистика | Финансы | Задачи | Проблемы | План на завтра
```

Значения — краткие (1-2 предложения) выжимки из каждого раздела.

---

## Правила интерпретации

- **Нетто отрицательное** → флаг 🔴 ПРИОРИТЕТ, добавить в "Проблемы"
- **Просроченные задачи > 3** → флаг ⚠️, перечислить
- **Алерты критичности 4-5 открыты** → обязательно в ТОП-3 проблем
- **Урожай = 0** → проверить данные или отметить как "нет урожая на этой неделе"
- **Нет данных за период** → явно указать "данные не зафиксированы"

