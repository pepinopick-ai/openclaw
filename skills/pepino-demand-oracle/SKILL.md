## 4 режима работы

### Режим 1 — Недельный прогноз спроса

Триггер: "сколько огурцов продадим на следующей неделе", "план на неделю"

```
Шаг 1: Считать данные из Sheets (🛒 Продажи — последние 4 недели)
Шаг 2: Применить сезонный коэффициент (таблица выше)
Шаг 3: Учесть активные заказы из CRM pipeline
Шаг 4: Вывести прогноз:

📈 ПРОГНОЗ СПРОСА — неделя [N]
Огурцы:      [X] кг  (база [Y] × сезон [k] = [Z] кг)
Вешенка:     [X] кг  ...
Шиитаке:     [X] кг  ...
Микрозелень: [X] кг  ...
Ферментация: [X] шт  ...
Confidence: [high/medium/low] (зависит от объёма исторических данных)
```

### Режим 2 — Производственный план под спрос

Триггер: "сколько сажать под следующий месяц", "план производства"

```
Алгоритм балансировки:

target_production = forecast_demand × (1 + safety_buffer)
где safety_buffer = 15% для скоропортящихся, 10% для ферментации

Учесть:
- Цикл огурцов: посев → harvest = 45–60 дней
- Цикл вешенки: инокуляция → harvest = 21–28 дней (1-й волна)
- Цикл шиитаке: инокуляция → harvest = 60–90 дней
- Цикл микрозелени: посев → harvest = 10–14 дней

Вывод:
📋 ПЛАН ПРОИЗВОДСТВА — [месяц]
Начать посев огурцов [N кг рассады] → ожидаемый урожай [дата]: [X кг]
Инокулировать [N блоков] вешенки → волна 1 [дата]: [X кг]
Посев микрозелени партии [N] [дата] → harvest [дата]: [X кг]
```

### Режим 3 — Балансировка остатков и заказов

Триггер: "остатки vs заказы", "что в избытке", "что дефицит"

```
Шаг 1: Считать остатки из Sheets (лист Склад / Производство)
Шаг 2: Считать открытые заказы из CRM pipeline
Шаг 3: Рассчитать баланс:

balance = stock + expected_harvest_7d - confirmed_orders_7d

ИТОГ:
🟢 Surplus [X кг] — рассмотреть pepino-innovation-lab (ферментация, переработка)
🟡 Tight supply — нельзя брать новых крупных клиентов
🔴 Дефицит [X кг] — нужно срочно посеять / предупредить клиентов
```

### Режим 4 — Анализ ресторанного pipeline + ABC-сегментация

Триггер: "что покупают рестораны", "топ клиенты", "кто заказывает нерегулярно", "ABC клиентов"

```
ABC-сегментация (по выручке за последние 90 дней):
  A-клиенты — топ 20% по выручке → дают ~80% дохода
    → Гарантировать наличие продукта всегда
    → Первый доступ к новым SKU
    → Личный контакт еженедельно (pepino-sales-crm)

  B-клиенты — следующие 30% → дают ~15% дохода
    → Стабильные условия, рассматривать апсейл

  C-клиенты — 50% → дают ~5% дохода
    → Обслуживать только при наличии излишков
    → Рассмотреть минимальный заказ или offboarding

Поведенческие сегменты:
  🏆 Champions:  заказы каждую неделю + растущий чек
  ✅ Loyal:       стабильные, нет роста
  ⚠️ At risk:    были частые, сейчас реже (>2 нед без заказа)
  🆕 New:        <3 заказов за историю
  ❌ Churned:    >4 недель без заказа → alert pepino-sales-crm

Вывод → передать список pepino-sales-crm для follow-up
```

### Режим 5 — Недельный пульс продаж

Триггер: "итоги недели по продажам", "как идут продажи", "выполнен ли план"

```
Читать из Sheets 🛒 Продажи за текущую и прошлую неделю:

  attainment_% = actual_revenue / planned_revenue × 100

  🟢 >90% — план выполнен
  🟡 70–90% — недовыполнение, есть причина
  🔴 <70% — критично, требует CEO-решения

Формат:
📊 ПУЛЬС ПРОДАЖ — неделя [N]
Факт: $[X] / План: $[Y] → [Z]% [🟢/🟡/🔴]
Топ SKU: [огурец X кг, грибы Y кг]
Новые клиенты: [N]
Следующая неделя pipeline: $[X] подтверждено
→ Сохранить в Sheets ⚙️ KPI
```

## Протокол алерта дефицита/переизбытка

При дисбалансе >20% — обязательно записать в Sheets:

```
Дефицит >20%:
  → ⚠️ Алерты: тип=производство, источник=pepino-demand-oracle,
    описание="Дефицит [SKU]: прогноз [X] кг vs доступно [Y] кг",
    критичность=4, статус=открыт
  → 📋 Задачи: "Срочно посеять/инокулировать [N] [единиц] к [дата]"

Переизбыток >30%:
  → Передать pepino-innovation-lab (ферментация, waste-to-value)
  → Предложить A-клиентам временную скидку (через pepino-sales-crm)
```
