### Алгоритм

```
ШАГ 1: Получить текущие курсы (через pepino-finance-tools или dolarapi.com)
  → Official (Oficial BNA)
  → Blue (параллельный)
  → MEP / Bolsa
  → CCL
  → Cripto (USDT)

ШАГ 2: Рассчитать спреды
  → Blue premium vs Oficial = (Blue - Oficial) / Oficial × 100%
  → MEP premium vs Oficial
  → Cripto premium vs Oficial

ШАГ 3: ТРИГГЕР — если Blue/Oficial разница > 5% от средней за 7 дней
  → Выдать 🔴 FX АЛЕРТ

ШАГ 4: Рекомендация закупок
  → Если Blue растёт > 3% за неделю → предложить ускорить USD-закупки
  → Если Blue стабилен или падает → оптимально ARS-операции
  → Если разрыв Blue/MEP > 15% → флаг регуляторного риска
```

## Модуль 2 — Dynamic Pricing Engine

**Триггеры вызова:**

- "инфляция", "пересмотр цен", "обновить прайс", "защита маржи"
- "сколько поднять цену", "как считать цену при инфляции"
- "dynamic pricing", "ценообразование"

### Модель ценообразования

```
Формула цены:
  Price = CostBase_ARS × (1 + InflationAdjustment) × (1 + DesiredMargin)

Где:
  CostBase_ARS     = фактическая себестоимость единицы (из unit economics)
  InflationAdjust  = накопленная инфляция с момента последнего обновления цены
  DesiredMargin    = целевая маржа (обычно 40-60% для Pepino Pick)
```

### Данные для расчёта

**Источники инфляции:**

- INDEC (официальный CPI): `https://www.indec.gob.ar`
- Прокси: Blue Dollar динамика за период

**Источники себестоимости:**

- `pepino-finance-tools unit [product]` — актуальная unit economics
- Лист "Финансы" — реальные расходы за период

### Алгоритм Dynamic Pricing

```
ШАГ 1: Определить продукт (veshyonka / microgreens / pepino / fermentado)

ШАГ 2: Получить текущую себестоимость
  → node finance.js unit [product]
  → Взять COGS из output

ШАГ 3: Рассчитать инфляционный коэффициент
  → Месяцев с последнего обновления цены: N
  → Среднемесячная инфляция: ~[текущий уровень, исторически 5-8%/мес]
  → InflationFactor = (1 + monthly_inflation)^N

ШАГ 4: Рассчитать новую цену
  → MinPrice = COGS × InflationFactor × 1.40   (40% margin minimum)
  → TargetPrice = COGS × InflationFactor × 1.55  (55% margin target)
  → PremiumPrice = COGS × InflationFactor × 1.70  (chef/premium accounts)

ШАГ 5: ТРИГГЕР — если инфляция накопленная > 20% с последнего обновления
  → Автоматически предложить пересмотр цен (не применять без approval!)

ШАГ 6: Сравнить с рынком
  → mercadolibre-analyst: текущие цены конкурентов
  → Позиционирование: ниже / на уровне / premium
```

## Модуль 3 — Inflation-Adjusted Margin Reporter

**Триггеры:** "реальная маржа", "маржа с учётом инфляции", "Real Margin"

### Расчёт Real Margin

```
Nominal Margin = (Revenue - COGS - OPEX) / Revenue

Real Margin (inflation-adjusted) =
  Пересчитать COGS и OPEX к текущим ценам (с InflationFactor)
  → Real COGS = Recorded_COGS × InflationFactor
  → Real OPEX = Recorded_OPEX × InflationFactor
  → Real Margin = (Revenue - Real_COGS - Real_OPEX) / Revenue
```

### Вывод в еженедельный review

Этот модуль автоматически добавляет секцию в `pepino-weekly-review`:

```
💹 РЕАЛЬНАЯ МАРЖА (с поправкой на инфляцию)
  Номинальная маржа:        [X]%
  Инфляция за период:       +[Y]%
  Реальная маржа:           [Z]%  [▲/▼ vs прошлая неделя]

  [🔴 если Real Margin < 25%: ALERT — маржа сжимается реально]
  [🟡 если Real Margin 25-35%: мониторинг]
  [🟢 если Real Margin > 35%: норма]
```
