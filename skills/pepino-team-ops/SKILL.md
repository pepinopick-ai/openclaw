# 👥 Pepino Pick — Team Operations

Управление командой, задачами, делегированием, HR и встречами. Снижает операционную нагрузку на CEO.

**Google Sheets ID:** `1AB9nkHfCu8_12dwn72tWKZOiqHAfAhI3AHOjeAAByoc`

- **Задачи** — task management
- **HR** — команда, роли, зарплаты
- **Стратегия** — OKR и цели
- **Решения** — фиксация решений встреч

## 5 режимов

### Режим 1 — Task Assignment & Delegation (Задачи и делегирование)

**Умное делегирование** — для каждой задачи определить:

- Может ли CEO делегировать? ДА / НЕТ
- Кому назначить (по роли и нагрузке)
- Приоритет: critical / high / medium / low
- Дедлайн

**Карта делегирования:**

```
ТОЛЬКО CEO (никогда не делегировать):
  → Ключевые клиенты (рестораны tier-1)
  → Решения о найме
  → Стратегические партнёрства и договоры
  → Капитал / банки / инвесторы
  → Публичные заявления и PR

МОЖНО ДЕЛЕГИРОВАТЬ (с контролем):
  → Координация сбора урожая → farm_manager
  → Планирование доставок → logistics_person
  → Посты в Instagram → marketing_person
  → Проверки качества → qa_person
  → Рутинные заказы у поставщиков → procurement_person
  → Ввод данных в таблицы → admin_person

АВТОМАТИЗИРОВАТЬ (без человека):
  → Еженедельный дайджест → pepino-weekly-review
  → Мониторинг FX → pepino-argentina-finance
  → QA алерты → pepino-qa-food-safety
  → Контроль бюджета → pepino-controller
```

**Запись задачи в "Задачи":**

```
Дата | Задача | Описание | Кому | Дедлайн | Приоритет | Статус | Контекст
```

**Формат создания задачи:**

```
✅ ЗАДАЧА СОЗДАНА
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Задача:       [название]
Кому:         [имя / роль]
Дедлайн:      [дата]
Приоритет:    [critical/high/medium/low]
Делегируемо:  [ДА/НЕТ] — [обоснование]
Контекст:     [что нужно знать исполнителю]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Режим 2 — Team Roster & HR

**Трекинг команды** (лист "HR"):

- Имя, роль, дата начала, зарплата ARS, расписание, ключевые навыки
- Ключевые обязанности
- История повышений / изменений

**HR операции:**

Онбординг нового сотрудника (чеклист):

```
☐ День 1: Знакомство с командой и фермой
☐ День 1: Доступ к Google Sheets (только нужные листы)
☐ День 1-3: Ознакомление с СОПами (через pepino-knowledge)
☐ Неделя 1: Shadowing — работа рядом с опытным сотрудником
☐ Неделя 2: Самостоятельное выполнение базовых задач
☐ Месяц 1: Ревью — что понял, что непонятно, что улучшить
☐ Месяц 3: Performance review + корректировка зарплаты если нужно
```

**Трекинг отпусков и отсутствий:**

```
Дата | Сотрудник | Тип (отпуск/больничный/отсутствие) | Дней | Замена
```

**Ревью зарплат** (раз в квартал):

- Сравнить с рынком (ARS с учётом инфляции)
- Учесть KPI за период
- Рекомендация: оставить / повысить / пересмотреть роль

### Режим 3 — Meeting Management

**Типы встреч Pepino Pick:**

| Встреча            | Частота                      | Длит.     | Цель            |
| ------------------ | ---------------------------- | --------- | --------------- |
| Daily standup      | Ежедневно (если команда > 2) | 15 мин    | Синхронизация   |
| Еженедельный обзор | Понедельник                  | 60 мин    | Итоги + фокус   |
| Месячный review    | Раз в месяц                  | 2 ч       | KPI + проблемы  |
| Ad-hoc             | По ситуации                  | 30–60 мин | Решение проблем |

**Шаблон заметок встречи:**

```
📋 ВСТРЕЧА — [тип] — [дата]
Участники: [список]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Обсудили:
  1. [тема]
  2. [тема]

Решения:
  → [решение] | Кто: [имя] | Дедлайн: [дата]
  → [решение] | Кто: [имя] | Дедлайн: [дата]

Открытые вопросы (не решены):
  → [вопрос] — решить до [дата]

Следующая встреча: [дата и тип]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

Решения → автоматически в лист "Решения"
Задачи из встречи → автоматически в лист "Задачи"

### Режим 4 — Workload Monitor (Нагрузка команды)

Еженедельный снимок нагрузки по каждому сотруднику:

- Открытые задачи: N
- Просроченные задачи: N
- Новые задачи за неделю vs закрытые (ratio)
- Алерт: если > 10 открытых задач → предложить перераспределение

**Формат отчёта:**

```
📊 НАГРУЗКА КОМАНДЫ — [дата]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Сотрудник     Открытые  Просроченные  Статус
[имя]:        [N]       [N]           [✅/🟡/🔴]
[имя]:        [N]       [N]           [✅/🟡/🔴]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Итого:        [N]       [N]

[🔴 ПЕРЕГРУЗКА: список сотрудников с > 10 задачами]
[Рекомендация: что перераспределить]
```

**Пороги:**

- > 5 открытых задач → 🟡 MONITOR
- > 10 открытых задач → 🔴 OVERLOAD → предложить перераспределение
- > 3 просроченных → 🟡 требует внимания

### Режим 5 — OKR & Goal Tracking

**Квартальные цели (OKR-light):**

Структура:

```
🎯 OKR Q[N] — [год]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Objective: [качественная цель]

Key Results:
  KR1: [измеримый результат] — [X]% выполнено
  KR2: [измеримый результат] — [X]% выполнено
  KR3: [измеримый результат] — [X]% выполнено

Ответственный: [кто]
Статус: on_track / at_risk / off_track
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

Читать/писать из листа "Стратегия".
Еженедельная проверка: на треке ли квартальные OKR?
